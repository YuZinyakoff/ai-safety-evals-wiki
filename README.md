# AI Safety & Evals Wiki

Это рабочая база знаний по курсу AI Safety & Evals.
Ее задача — постепенно превращать материалы курса из `raw/` в поддерживаемый слой знаний в `wiki/`.

## Что здесь хранится

### 1. Исходники
Папка `raw/` содержит исходные материалы курса:
- теоретические тексты
- ноутбуки с практикой
- дополнительные материалы

### 2. Wiki
Папка `wiki/` содержит поддерживаемую базу знаний:
- страницы по неделям
- страницы по отдельным источникам
- страницы по концептам
- синтезы и сводки

Содержимое wiki по умолчанию ведется на русском языке.

## Логика работы

1. Новые материалы складываются в `raw/`
2. `raw/` остается источником истины
3. По возможности каждый источник сначала доводится до локально доступного текста, но не любой локальный текст одинаково хорош:
   - предпочитается качественный clipped `.md`
   - для arXiv обычно лучше всего clipped `.md` из HTML-версии, при сохранении PDF как визуального оригинала
   - sidecar `filename.ext.md` рядом с бинарным оригиналом допустим как fallback-extract, но не считается автоматически лучшей версией
   - URL-only источник считается временно неполным состоянием
4. Для формул, таблиц и сложной верстки при необходимости полезно держать рядом `TeX Source` или другой более точный исходник
5. Если sidecar из PDF явно хуже уже существующего clipped/HTML/TeX текста, его не нужно держать в `raw/` только ради “полноты”
6. Codex помогает выбрать лучшее представление источника и встроить материал в `wiki/`
7. Wiki постепенно становится основным слоем для поиска, повторения и синтеза
8. Полезные ответы и сводки сохраняются обратно в `wiki/`

Для source pages важно сохранять не только общий тезис источника, но и короткую карту его структуры: как устроен аргумент, paper или notebook. Это особенно полезно для больших статей, к которым потом хочется возвращаться как к учебным текстам, а не только как к списку выводов.

Пустые source pages не создаются заранее: если в `raw/` еще нет материалов, поддерживаются только обзорные страницы wiki.

## Структура

- `raw/week-01..week-05/` — материалы по неделям
- `raw/shared/` — материалы вне недель
- `wiki/weeks/` — страницы недель
- `wiki/sources/` — страницы источников
- `wiki/concepts/` — ключевые концепты
- `wiki/syntheses/` — более крупные сводки

## Основной workflow

- capture: добавить новый raw-материал
- normalize: по возможности получить лучший доступный локальный текст источника в `raw/`
- ingest: встроить материал в wiki
- query: ответить по базе знаний
- lint: почистить и укрепить структуру wiki
